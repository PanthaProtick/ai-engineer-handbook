# Understanding the Web

## Client
Clients are the typical web user's internet-connected devices.

## Server
Servers are computers that respond to client requests.

## TCP/IP
Transmission Control Protocol and Internet Protocol (TCP/IP) are communication protocols that define how data travels across the internet.

## DNS
The Domain Name System (DNS) is like an address book for websites. It stores the actual address of servers.

## HTTP
Hypertext Transfer Protocol (HTTP) is an application protocol that defines the language clients and servers use to communicate.

## What Happens Exactly
When you type a web address, which is technically part of a URL, into your browser's address bar, the following steps occur:

1. The browser queries a DNS server and finds the real address of the server hosting the website.
2. The browser sends an HTTP request to the server, asking for a copy of the website. This message, and all other data sent between the client and the server, travels across your internet connection using TCP/IP.
3. If the server approves the client's request, it sends a `200 OK` response, then starts sending the website's files to the browser as a series of small chunks called packets.
4. The browser assembles the chunks into a complete web page and displays it to you.

## Packets
When data is sent across the web, it is broken into small chunks called packets. Each packet contains:

1. A header, which includes details such as the server and client IP addresses, the packet number, the total number of packets in the transmission, and the protocols used.
2. A payload, which contains the actual data sent in the packet.

## HTTP Methods

### GET
Retrieves a specific resource, such as an HTML file containing product information or a list of products.

### POST
Creates a new resource, such as a new article in a wiki or a new contact in a database.

### HEAD
Retrieves metadata about a resource without returning the body, which is useful for checking when a resource was last updated.

### PUT
Updates an existing resource, or creates a new one if it does not exist.

### DELETE
Deletes the specified resource.

## URL Parameters
GET requests encode data in the URL sent to the server by adding name/value pairs onto the end of it — for example http://example.com?name=Fred&age=11. You always have a question mark (?) separating the rest of the URL from the URL parameters, an equals sign (=) separating each name from its associated value, and an ampersand (&) separating each pair. URL parameters are inherently "insecure" as they can be changed by users and then resubmitted. As a result URL parameters/GET requests are not used for requests that update data on the server.

## Client-side cookies
Cookies contain session data about the client, including keys that the server can use to determine their login status and permissions/accesses to resources.

## Static Sites
A static site is one that returns the same hard coded content from the server whenever a particular resource is requested.

## Dynamic Sites
A dynamic site is one that can generate and return content based on the specific request URL and data.
