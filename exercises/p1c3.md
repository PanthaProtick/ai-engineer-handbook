# Explain HTTP in your own words.

HTTP is a language format for client and server applications to communicate regardless of their variety.

# Why does HTTP exist?

To imply a set of rules while communicating between devices via web communication.

# Explain the difference between: GET, POST, PUT, PATCH, DELETE using your own examples.

GET-> request to fetch resources (Ex: Fetch a webpage)
POST-> send data to generally make a new entry or changes on database (Ex: Create a user with email and password)
PUT-> replace the whole entry with provided data fields (Ex: Replace all the information of a user with provided data)
PATCH-> instead of replacing the whole entry like PUT just patch the provided fields (Ex: only change the email of the user)
DELETE-> simply delete associated entry from the database (Ex: Delete a post containing provided id)

# Break down this request:

POST https://api.example.com/users/5?admin=true

Protocol->https
Domain->api.example.com
Path->/users/5
Query Parameter->admin=true
Method->POST

# Explain these status codes:

200->When a server successfully sends back a response
201->When a new user is created
400->When a user sends a syntactically invalid request
401->When a user is logged in
403->When a user is not allowed to access a resource
404->When there is no resource for request
500->When server crashes
